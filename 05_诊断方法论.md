# 05 诊断方法论：取证式定位根因

## 核心定义

审查纠偏防的是**错误进入**；诊断方法论查的是**已发生的错误**。两者是姊妹环节：一个在交付前拦截，一个在缺陷暴露后定位。

诊断最大的失效模式不是"查不出来"，而是"只看了 bug 对应的那一小块，没采集周边证据就开方"——修了受害者，放走了真凶，还波及了无辜。

所以诊断的核心纪律是：

> 先取证，后下结论；诊断的人不动手术。

## 1. 三条纪律

### 1.1 先取证后结论

不在读完证据前猜根因。开局可以有"根因假设"，但必须显式标注**待证实**，并允许随时被推翻。

### 1.2 现象 ≠ 根因

必须显式区分"用户看到的现象"和"真正出问题的那一层"。现象暴露点往往只是**受害者**——坏数据、坏状态从上游流下来，在这里刚好可见。在受害者身上打补丁是治标。

### 1.3 每条结论回指证据

诊断报告中的每个论断，都要能回指一次真实的工具动作（读过的文件、跑过的 grep、查过的 git 历史）。指不回去的，标注为 **[假设，未证实]**，不得伪装成事实。

## 2. 六维侦察

对每个 bug，先快速过一遍六个维度。每个维度都要做**真实工具动作**并落地产物，不能凭印象：

| 维度 | 动作 | 产物 |
|---|---|---|
| ① 现象确认 | 用自己的话复述 bug，分清现象 / 期望 | 一句话根因假设（标"待证实"） |
| ② 定位锚点 | 搜索/读代码，确认现象暴露的精确位置 | file:line 锚点（读过才算） |
| ③ 根因上溯 | 从锚点沿调用链往上读 | 坏数据/坏状态来自**哪一层** |
| ④ 爆炸半径 | grep 待改符号的所有调用方/消费方 | 改动会波及谁 |
| ⑤ 横向兄弟 | 搜索同模式的其他实例（同类页面/组件/函数） | 兄弟是否同病 / 是否有正确范本 |
| ⑥ 历史意图 | git blame / log 相关行 | 这段代码当初**为什么长这样** |

维度 ④⑤ 是广撒网搜索，可以派检索型子 agent 并行；②③⑥ 留在主上下文——诊断靠的是**跨维度连点成线**，不能交给互相看不见的子 agent。

## 3. 自适应深挖：四个触发条件

侦察后，满足任一触发条件就深挖对应维度；都不满足，轻 bug 到此为止——这是"自适应"的关键，不为简单问题付重流程成本。

| 触发 | 条件 | 深挖动作 |
|---|---|---|
| A 根因不在锚点 | 现象点只是受害者，坏数据来自上游 | 沿调用链逐层上溯，直到找到**引入**坏状态的那一层；在它之前的任何"修复"都是治标 |
| B 存在兄弟 | 发现是共享模式的一个实例 | 逐个读兄弟：同病则修复须覆盖全部；有正确范本则照抄它 |
| C 爆炸半径大 | 调用方多、被广泛依赖 | 逐个评估调用方是否依赖旧行为，列出会被打破的调用点 |
| D 历史显示有意为之 | git 历史表明这段代码是为修另一个问题加的 | 查清当初意图，评估改回去会不会**回归**老 bug |

深挖中随时可以推翻侦察阶段的根因假设。反复深挖直到**根因唯一、证据闭环**。

## 4. 输出：根因分析 + 候选方案

### 4.1 根因分析

```text
现象：<用户看到的>
期望：<应该怎样>
根因：<真正出问题的那一层，精确到 file:line>
  ↑ 证据：<哪个维度 / 哪次读取支撑了它>
现象 → 根因链路：<一两句话讲清因果链>
关联面：爆炸半径 / 横向兄弟 / 历史回归风险（或"无"）
```

### 4.2 候选方案：有真分叉才多个

给出 1–3 个候选方案。**只有存在真实分叉时才给多个**（治标快修 vs 治本重构、改公共逻辑 vs 各兄弟单独改）；只有一条理性路径时就给一个，不硬凑假选项。

每个方案写清：思路、改动范围（每条回指一条证据）、取舍（治标/治本、改动大小）、风险（可能引入的新问题、回归点）。多方案时给一句推荐 + 理由。

### 4.3 停下，交回选择权

**诊断的人不动手术。** 呈现分析与方案后停下，由人选择：采纳某方案进入计划流程（随后走 plan review）、补充信息重新诊断、或继续深挖某个维度。绝不自动开工——诊断与修复分离，是防止"带着锤子找钉子"的结构性保障。

## 5. 诊断产物的去向

- 诊断报告持久化到项目计划目录（如 `plan/diagnoses/`），作为后续计划阶段的**输入凭据**——计划直接引用诊断证据，避免执行时重新跑偏。
- 诊断中发现的兄弟同病、历史债，本次不修的登记技术债。
- 反复出现的同类根因，走规则升级（见《技术债与经验回流》）。

## 6. 诊断检查清单

- [ ] bug 已复述，现象与期望分清？描述含糊时先问人，不带歧义往下查？
- [ ] 六维各有真实工具动作与产物？
- [ ] 根因假设是否被证据推翻过至少一次质疑？
- [ ] 每条结论可回指证据？未证实的标了 [假设]？
- [ ] 触发条件核过，该深挖的深挖了？
- [ ] 候选方案是真分叉，不是凑数？
- [ ] 停在了选择权交回处，没有顺手开修？
