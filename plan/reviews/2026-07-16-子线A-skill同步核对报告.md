## 差异清单（子线原始发现）

**1. [change-review.md] REQUEST CHANGES verdict 语义与 KB 04 §4.2 不一致（机制级）**
- skill：APPROVE WITH CHANGES = 存在阻塞/重要问题（修完可交付）；REQUEST CHANGES = 需重新思考的根本性设计问题，**不走增量、必须全量重审**。
- KB 04 §4.2 原文：REQUEST CHANGES =「存在阻断问题，不能交付」，且示例列的全是常规可修缺陷（旧 path 未切换/测试缺覆盖/README 失效）。
- 影响：同一枚举值的严重度门槛不同，直接影响"是否全量重审"的闭环走向。

**2. [change-review.md + commit.md] history 归档行丢失 status 字段（机制级）**
- KB 04 §10.2 与 B 模板库承诺：归档保留 `resolved|accepted|wontfix`，"状态比删除可靠"。
- 两个 skill 的归档格式实际只含 CATEGORY/description/severity——归档后 ACCEPTED/WONTFIX 与已修复无法区分。
