# 子线 B 报告 — KB 仓机器闸门实施（2026-07-16）

> 派发背景：KB 优化计划 P0——修 check_sync.sh、新建交叉引用校验、装 pre-commit 闸门。两轮交付（首轮 + 一条 must-fix 迭代），主会话已验收。

## 交付物

1. **`plan/check_sync.sh`（修复）**
   - 乱码根因：macOS `/bin/bash` 3.2 多字节 bug——`$file` 后紧跟全角括号「（」时，bash 把「（」首字节 EF 吃进变量名，展开为不存在的变量（空值）+ 残留孤立字节显示为 ��。修复：所有含变量的中文输出改 printf 格式串，变量永不与多字节字面量相邻（头注已写明防复发）。
   - 自证执行：结尾输出「已核对 N 条」，N==0 报错退出 1；skill 缺失/基线缺 md5/md5 输出为空均显式报错。bash 3.2 兼容。
2. **`plan/check_refs.sh`（新建）**
   - 校验三类引用：`《章名》§N[.N]`（章名→文件用**分隔符归一化包含匹配**：剥 、，, 空格 _ - + ASCII 小写化；0/多命中走 warning；一级 0 命中时二级兜底剥连接词「与」）、`[xx.md](./xx.md) §N[.N]`、`案例库 #N`。
   - 违规输出 文件:行号+原文+原因；「已核对 N 处（违规 X，无法判定 Y）」，N==0 报错。
3. **`.githooks/pre-commit`（新建）**：串联两闸门，全跑完再汇总，任一失败拒绝提交，提示 `--no-verify` 逃生；`git config core.hooksPath .githooks` 已生效。
